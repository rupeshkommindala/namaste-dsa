# Time Complexity and Space Complexity
# The algorithm which has low Time Complexity, that means it has better speed
# Time Complexity is not equal to the Time taken by the Algorithm
# Time Complexity is the best way to measure the efficiency of an Algorithm
# Time taken is not the right way to define the efficiency of an Algorithm
# Low Time Complexity => Better Speed 
# Binary Search >>>> Linear Search
# Big O is just a way to represent the worst case Time Complexity of an Algorithm
# O(1)>O(logn)>O(n)>O(nlogn)>O(n^2)>O(2^n)>O(n!)
# The more left we go the better the Time Complexity will be
# Notation is nothing but a symbol
# Time Complexity: It is used to measure the efficiency of an algorithm in terms of speed as the input size grows
# Space Complexity: How much extra space your algorithm is using
# Constant Time Complexity: If the array size is 100, still this will take only one operation
#                           If the array size is 1000, still this will take only one operation
